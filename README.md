## Análise da aplicação

### Objetivo da aplicação

A aplicação tem como objetivo permitir o cadastro e a visualização de cursos. 
O usuário pode preencher um formulário para registrar um novo curso e visualizar os cursos cadastrados em uma lista.

### Principais fluxos da aplicação

Durante a exploração da aplicação foram identificados os seguintes fluxos principais:

1. Cadastro de curso
O usuário preenche os campos do formulário e realiza o cadastro de um novo curso.

2. Listagem de cursos
Após o cadastro, os cursos registrados são exibidos em uma lista na aplicação.

3. Validação de dados
O sistema deve validar os campos obrigatórios antes de permitir o cadastro do curso.

### Pontos críticos para teste

Durante a análise foram identificados alguns pontos críticos que devem ser priorizados nos testes:

- Validação dos campos obrigatórios no cadastro de cursos
- Funcionamento correto do botão de cadastro
- Exibição correta dos cursos cadastrados na listagem
- Comportamento da aplicação em cenários negativos (campos vazios ou inválidos)
